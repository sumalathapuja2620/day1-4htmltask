WHAT IS STRUCTURE OF URL?

A URL, or Uniform Resource Locator, is an address on the internet that identifies a resource. The structure of a URL is made up of several components, including the protocol, domain, and path.

Components of a URL:-

*Scheme:The protocol used to access the resource, such as HTTP or HTTPS.

*Domain: name of the host that contains the resource, such as www.example.com.

*Path: location of the resource on the host, such as /software/htp/cics/index.html.

*Query string:A string of information that the resource can use, such as search parameters.

*Sub-Domain A part of the domain that comes before the main domain.

*Top-level domain (TLD):The domain extension, such as .com, .org, or .net.

*Port: The network port used to connect to the resource .

*Fragment:A reference point within a file, such as an anchor in an HTML file.

HOW THE WORKS URL?

When you enter a URL into a browser, the browser looks up the resource's IP address and sends a request to the server that hosts it. The server responds by sending the requested resource back to the browser. The browser then renders the resource as a web page.

Steps:-

1.Enter the url:Type the URL into the browser's address bar.

2.Find the IP address:The browser uses a DNS lookup to find the IP address of the domain.

3.Establish a connection:The browser establishes a TCP connection with the server.

4.Send the request: The browser sends an HTTP request to the server.

5.Receive the response: The server processes the request and sends back a response.

6.Render the page:The browser renders the HTML, CSS, and JavaScript files in the response to display the page.

AWS and AZURE:

AWS (Amazon Web Services) and Azure (Microsoft Azure) are both cloud computing platforms that provide a wide range of services like compute power, storage, databases, networking, and more, allowing users to access and manage their applications and data online without needing to manage physical servers, essentially functioning as a virtualized infrastructure where you can rent computing resources as needed; while they offer similar capabilities, they differ in architecture, pricing models, and integration with their respective company's ecosystem, with AWS generally considered to have a larger market share and Azure offering tighter integration with Microsoft products like Office 365 and Windows Server.

Key points about how AWS and Azure work:-

Core services:

Both platforms offer core services like virtual machines (EC2 instances on AWS, virtual machines on Azure), storage (S3 on AWS, Blob Storage on Azure), databases (RDS on AWS, SQL Database on Azure), and networking capabilities (VPCs on AWS, virtual networks on Azure).

Access and Management:

Users access and manage their cloud resources through a web console or APIs, allowing them to provision servers, configure networks, deploy applications, and manage data storage.

Scalability:

Both platforms are designed to scale up or down based on demand, meaning you can easily add or remove computing power as needed without having to manually manage hardware.

Hybrid Cloud:

Both AWS and Azure support hybrid cloud environments, allowing users to integrate their on-premise infrastructure with cloud resources.

Key differences:-

Integration with Microsoft Ecosystem:

Azure is tightly integrated with Microsoft products like Office 365 and Active Directory, making it particularly attractive for businesses heavily reliant on Microsoft services.

Pricing Model:

AWS generally offers a more flexible pricing model with various options for different usage scenarios, while Azure can sometimes have a more straightforward pricing structure.

User Interface:

Some users find Azure's interface to be more user-friendly, especially for those familiar with Microsoft products, while AWS may have a steeper learning curve for new users.

SOFTWARE DEVELOPMENT LIFE CYCLE(SDLC):

The Software Development Life Cycle (SDLC) is a process that describes the stages of developing software. It includes planning, design, development, testing, deployment, and maintenance.

SDLC phases:-

1.PlanningThe initial stages of the project, including defining requirements.

2.DesignThe process of creating the software's design.

3.DevelopmentThe process of building the software.

4.TestingThe process of evaluating the software's functionality.

5.DeploymentThe process of delivering the software to users.

6.MaintenanceThe process of keeping the software updated and running.

SDLC Models:-

*Spiral ModelA risk management model that combines iterative development with Waterfall elements. It's often used for large, complex projects.

*Agile ModelA model that uses small, incremental changes to respond to changing requirements. It's often used to accelerate production.

*Iterative Model model that uses small cycles to make frequent changes and avoid mistakes. It's often used for large projects with clearly defined requirements.
